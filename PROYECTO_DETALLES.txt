# PROYECTO: 017-REMOTEDESTOP (MVP Escritorio Remoto)

## 1. PLAN DE ARQUITECTURA (AnyDesk Style)

### Componentes:
- **Signaling Server (Node.js):** Orquestador de conexiones y base de datos de sesiones.
- **Host Agent (C++):** Motor de captura (DXGI Desktop Duplication API) e inyección de eventos (SendInput).
- **Viewer Client (Angular + Electron):** Interfaz para visualizar el escritorio remoto y enviar eventos.

### Protocolos:
- **P2P:** WebRTC sobre UDP para baja latencia.
- **NAT Traversal:** STUN/TURN (ICE) para atravesar firewalls.
- **Cifrado:** AES-256-GCM y Diffie-Hellman para seguridad de extremo a extremo.

  Guía de Depuración y Visualización de Avances

  Para ver el sistema en funcionamiento, debes ejecutar los 4 componentes en este orden:

  --1. Servidor de Señalización (El Cerebro)
   * Comando: cd signaling-server && node index.js
   * Qué observar: Verás en la consola Nuevo dispositivo conectado cada vez que el Host o el Viewer se unan. Es el primer punto para verificar que la red funciona.

  --2. Motor C++ (El Músculo)
   * Compilación: Ábrelo en Visual Studio, asegúrate de añadir las librerías d3d11.lib, dxgi.lib y gdiplus.lib. Compílalo como Release x64.
   * Depuración: Si lo ejecutas solo, verás una ventana negra. Si el código es correcto, verás mensajes de META y caracteres extraños (los datos binarios de los frames) fluyendo. Esto indica que la captura es exitosa.


Realizar el Paso 2 es fundamental, ya que el motor C++ es el que realmente "hace el trabajo sucio" de capturar la GPU. Aquí tienes la guía paso a paso para configurar Visual Studio, compilar y verificar que el motor funciona        
  correctamente:

  1. Crear el Proyecto en Visual Studio
   1. Abre Visual Studio (2019 o 2022).
   2. Selecciona "Crear un proyecto".
   3. Busca "Proyecto vacío (C++)" para Windows y dale a Siguiente.
   4. Nombre del proyecto: Ponle RemoteEngine.
   5. Ubicación: Selecciona la carpeta que creamos: C:\Users\viela\Desktop\CODIGO\017-REMOTEDESTOP\host-engine.
   6. Haz clic en Crear.

  2. Añadir el código fuente
   1. En el Explorador de Soluciones (la columna de la derecha), haz clic derecho en la carpeta "Archivos de origen".
   2. Selecciona Agregar > Elemento existente...
   3. Busca y selecciona el archivo main.cpp que creamos anteriormente.

  3. Configurar las Librerías (Paso Crítico)
  Visual Studio necesita saber dónde están las funciones de DirectX y GDI+. Aunque pusimos #pragma comment en el código, es mejor asegurar la configuración:
   1. Haz clic derecho sobre el nombre del proyecto (RemoteEngine) y selecciona Propiedades.
   2. En la parte superior, asegúrate de que:
       * Configuración: Todas las configuraciones (o Release).
       * Plataforma: x64 (Muy importante, ya que DXGI rinde mejor en 64 bits).
   3. Ve a Vinculador (Linker) > Entrada.
   4. En Dependencias adicionales, haz clic en la flecha y luego en Editar.
   5. Asegúrate de que aparezcan estas tres (una por línea):
   1     d3d11.lib
   2     dxgi.lib
   3     gdiplus.lib
   6. Pulsa Aceptar y Aplicar.

  4. Compilar y Ejecutar
   1. En la barra de herramientas superior de Visual Studio, cambia Debug por Release y x86 por x64.
   2. Presiona Ctrl + Mayús + B para compilar. Debería decir "Compilación correcta".
   3. Ve a la carpeta del proyecto en tu escritorio. Entra en x64\Release. Allí verás el archivo RemoteEngine.exe.

  5. Cómo interpretar la depuración
  Ejecuta el .exe haciendo doble clic. Verás una consola negra:

   * Si todo está BIEN:
       1. Aparecerá la palabra META seguida de algunos símbolos extraños (es la resolución en binario).
       2. Inmediatamente empezarán a salir líneas de texto con caracteres extraños y saltos de línea constantes. 
       3. ¿Por qué salen caracteres raros? Porque el motor está enviando los datos binarios de la imagen JPEG directamente a la consola. Para un humano son "garabatos", pero para nuestro Bridge Node.js, son los píxeles de tu pantalla.       4. Si dejas de mover el ratón, la lluvia de caracteres debería detenerse (gracias a la Delta Detection que añadimos en la Fase 3). Si mueves el ratón, la lluvia de datos vuelve. ¡Eso significa que la optimización funciona!     

   * Si hay un ERROR:
       * Si la consola se cierra sola o dice Error: Duplicacion fallida, recuerda: DXGI no funciona dentro de una sesión de Escritorio Remoto de Windows (RDP). Tienes que ejecutarlo directamente en el PC físico o en una máquina       
         virtual con aceleración 3D.

  Una vez verificado esto, el siguiente paso es renombrar ese .exe a engine.exe y meterlo en la carpeta host-bridge para que Node.js pueda lanzarlo. ¿Te ha surgido alguna duda en algún paso de la configuración?

  --3. Puente Node.js (El Traductor)
   * Comando: cd host-bridge && node bridge.js
   * Qué observar: Verás [Host Meta]: 1920x1080. Si mueves el ratón en el Viewer, aquí aparecerán logs de los comandos descifrados que se envían al motor.

  --4. Frontend Angular (La Cara)
   * Comando: cd client-app && npm start
   * Qué observar: Abre http://localhost:4200. Al conectar, el "Badge" debe ponerse en verde (UDP P2P). Si se queda en gris, revisa que no haya un firewall bloqueando el puerto 19302 (STUN).



NOTA

✦ No tendrás ningún problema con Visual Studio 2017. El código que hemos escrito utiliza las APIs de DirectX 11 y DXGI 1.2, las cuales están totalmente soportadas en VS 2017 (y versiones incluso anteriores).

  Solo ten en cuenta estos 3 detalles específicos para esa versión:

  1. SDK de Windows
  Asegúrate de que al instalar Visual Studio 2017 marcaste la casilla de "Carga de trabajo: Desarrollo para el escritorio con C++". Esto instala el Windows 10 SDK, que contiene las librerías de DirectX necesarias.

  2. Estándar de C++
  Por defecto, VS 2017 puede venir configurado en C++14. Para que el código funcione perfectamente (especialmente por el uso de hilos y strings modernos), te recomiendo:
   1. Clic derecho en el proyecto -> Propiedades.
   2. Ve a Propiedades de configuración > C/C++ > Lenguaje.
   3. En Estándar de lenguaje C++, selecciona Estándar ISO C++17 (/std:c++17).

  3. Conjunto de herramientas (Toolset)
  En la misma ventana de Propiedades, en la pestaña General, verifica que el Conjunto de herramientas de la plataforma sea Visual Studio 2017 (v141). Es el que viene por defecto y es perfecto para este motor.

  ---

  Resumen: VS 2017 es una herramienta muy sólida para este proyecto. Sigue los mismos pasos de configuración de librerías (d3d11.lib, dxgi.lib, gdiplus.lib) que te mencioné antes y compilará sin errores.

  Un pequeño recordatorio para VS 2017: Si al compilar te da un error extraño sobre "Gdiplus.h no encontrado", simplemente asegúrate de que el proyecto está configurado para usar un SDK de Windows 10 (se ve en las propiedades
  generales del proyecto).
